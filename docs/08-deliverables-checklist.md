# Academic Project Deliverables Checklist

Academic software projects should provide enough material for another person to install, run, inspect, and evaluate the work.

## Checklist

- Source code with a clear structure
- README with setup and usage commands
- Local knowledge base documents
- Test suite and evaluation command
- Test and evaluation report
- Explanation of model and retrieval choices
- Notes about privacy and local artifacts
- Screenshots or demo instructions

## Submission Advice

Before submission, run tests, run retrieval evaluation, rebuild the index after changing documents, and verify that generated files such as virtual environments and local databases are ignored by Git.
