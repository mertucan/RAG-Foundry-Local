# Project Report Requirements

The project report should explain what was built, why it was built, how it works, how it was tested, and what limitations remain. It should be readable by someone who has not followed the development process.

## Required Sections

1. Introduction and problem statement
2. System architecture
3. Data and document preparation
4. RAG pipeline explanation
5. User interface description
6. Testing and evaluation results
7. Limitations and future work
8. Conclusion

## Evidence

Include screenshots, command outputs, evaluation tables, and source references where appropriate. The report should connect implementation choices to project requirements.
